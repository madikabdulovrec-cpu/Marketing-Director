# Инструкция для Claude (любая сессия)

> Этот файл Claude Code автоматически загружает в контекст при старте.
> Поэтому держим его коротким и направляющим, без деталей.

## Первое действие в любой сессии

**Прочитай `memory/MEMORY.md`.** Это индекс всех проектов с указанием, где лежит код / какой статус / какой файл открыть для деталей.

После этого:
- Если пользователь назвал конкретный проект → открой соответствующую карточку `memory/project_*.md` и работай по ней.
- Если пользователь не назвал проект → спроси, над каким работаем (список — в MEMORY.md).

## Правила работы с этим репозиторием

1. **Карточки `memory/project_*.md` — это контракт между сессиями.** Если в проекте появилось новое решение, новый артефакт, новая договорённость — обнови карточку проекта **в той же сессии**. Без этого следующая сессия (или сессия на другом устройстве) не узнает.
2. **`.gitignore` защищает чувствительное** (брифы клиентов, бухгалтерия, входящие PDF, токены, бухгалтерские xlsx). При добавлении новых файлов проверяй: если контент конфиденциален — добавь правило в `.gitignore` до коммита.
3. **Receipts-бот живёт в своём репо** `github.com/Evimiz-KZ/cheki-bot`. В этом репо его кода нет — только карточка в `memory/project_receipts_bot.md`. Изменения по боту коммитятся туда, не сюда.
4. **Старые версии стратегий** не оставляй в репо — переноси в `.gitignore`. Актуальной должна быть одна версия документа.

## Когда заводится новый проект

1. Создай `memory/project_<slug>.md` по шаблону любой существующей карточки.
2. Добавь строку в таблицу в `memory/MEMORY.md`.
3. Добавь готовый «Запуск сессии» промпт в раздел MEMORY.md → «Как продолжить на другом устройстве».
