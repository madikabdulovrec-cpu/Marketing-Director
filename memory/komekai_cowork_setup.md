# KomekAI — настройка для Claude Cowork

> Как использовать KomekAI-контекст именно в Cowork, чтобы не копировать
> промпт каждый раз и подключить нужные скилы и коннекторы.

## Принцип Cowork (короткое объяснение)

В Cowork есть три уровня настройки, которые в Claude Code заменяются файлами `CLAUDE.md` + `memory/`:

1. **Project / Workspace context** — постоянная инструкция, загружается автоматически в каждой сессии. Аналог `memory/project_komekai.md`.
2. **Skills** — готовые слэш-команды для типовых артефактов (`/marketing:campaign-plan` и т.п.). Аналог — заранее заготовленные промпты в `memory/`.
3. **Connectors** — интеграции с Gmail, Calendar, Drive и др. Аналог — отдельные внешние сервисы.

Готовый `komekai_session_brief.md` работает как-есть в **любом** интерфейсе Claude — но в Cowork эффективнее настроить через project context + скилы.

---

## Шаг 1 — Project context (один раз)

Создай в Cowork новый проект **«KomekAI»** и вставь в его описание / custom instructions содержимое файла:

```
memory/komekai_session_brief.md
```

После этого в любой новой сессии этого проекта Claude уже знает контекст — не надо копировать промпт первым сообщением.

**Альтернатива:** если в твоём Cowork project context имеет лимит по длине — вставь сокращённую версию:

```
Работаем над KomekAI — голос+текст AI-агент для МСБ Казахстана.

Категория: автопилот клиентских коммуникаций и маркетинга (реклама → ответ
24/7 → запись). Обещание: «Ни одного пропущенного клиента». Архетип:
Caregiver + Sage. ICP P1: стоматологии, косметология, аренда коммерч. и
жилья (Алматы/Астана). ARPU 100к₸/мес.

Главные ограничения:
- Не обещай как готовое то, что в разработке (мессенджеры, биллинг)
- Не позиционируй как «замену сотрудников» и не используй «цифровой двойник»
- Не используй техно-жаргон (LLM, RAG) для МСБ
- Не предлагай скидки / «успей купить» — против Caregiver
- Не расширяй ICP за пределы P1 первые 3 мес

Полный контекст — komekai_session_brief.md и KomekAI_STP_Goals_v1.docx.
```

## Шаг 2 — Скилы под конкретные артефакты

В Cowork доступны marketing-скилы. Под каждый артефакт из бэклога KomekAI — рекомендованный скил:

| Артефакт | Скил | Пример команды |
|---|---|---|
| Операционный план Q1 | `/marketing:campaign-plan` | «Используй контекст KomekAI. Сделай 90-дневный план запуска в P1: стоматологии + косметология. Цели Q1 — из брифа.» |
| Лендинги (главная + 3 отраслевые) | `/marketing:draft-content` | «Используй контекст. Напиши тексты для главной + отраслевого лендинга «KomekAI для стоматологий». Tone — Caregiver, не техножаргон.» |
| Контент-план Q1 | `/marketing:content-creation` | «Контент-план Q1 для KomekAI на 3 канала: Telegram (отраслевые чаты), LinkedIn (B2B аутрич), YouTube Shorts. По 3 рубрики на канал.» |
| Outbound-аутрич по email | `/marketing:email-sequence` | «Sequence 5 писем для аутрича владельцам стоматологий Алматы. Триггер — открыли новый кабинет.» |
| Sales-демо скрипт | `/marketing:draft-content` | «Напиши 30-минутный sales-демо скрипт для встречи с главврачом стоматологии. Структура: проблема → продукт → live-демо → пилот.» |
| Проверка любого готового материала | `/marketing:brand-review` | «Прочитай этот текст лендинга и проверь на соответствие Caregiver + Sage. Найди техножаргон и фразы давления.» |
| Анализ конкурентов | `/marketing:competitive-brief` | «Сравни KomekAI с Vapi, Retell, Tinkoff Sage по 10 критериям. Найди gaps в позиционировании и messaging.» |
| Отчёт по результатам Q1 | `/marketing:performance-report` | «Подведи итоги Q1 по KomekAI: что закрыли в OKR, что нет, какие выводы.» |
| SEO для сайта | `/marketing:seo-audit` | «SEO-аудит будущего сайта KomekAI: ключевые запросы по «AI ассистент для клиники Алматы», конкурентная выдача, контентные gaps.» |

**Принцип:** запускаешь нужный скил → даёшь короткий контекст-якорь («используй контекст KomekAI» + 1-2 уточнения) → скил выдаёт артефакт в своём шаблоне. Тебе не надо писать длинный промпт каждый раз.

## Шаг 3 — Коннекторы (подключи, если делаем outbound)

Какие коннекторы реально пригодятся для KomekAI:

| Коннектор | Зачем для KomekAI |
|---|---|
| **Gmail** | Outbound-аутрич по списку 30 ЛПР P1. Скил `/marketing:email-sequence` → Gmail → отправка. Отслеживание ответов. |
| **Calendar** | Расписание демо со ЛПР, окна пилотов, weekly check-in с CSM-клиентами. |
| **Drive** | Расшаривание лендингов, кейсов, презентаций с командой. Хранение стратегии для команды без Claude-доступа. |
| **Slack** (если есть) | Уведомления о новых пилотах, MQL, NPS. |
| **HubSpot / Pipedrive** (если используешь) | CRM-синхронизация лидов из MQL → демо → пилот → платный. |

**Минимальный набор на старт:** Gmail + Calendar. Остальное — по мере роста sales-команды.

## Шаг 4 — Типичные сценарии работы в Cowork

### Сценарий А: новый артефакт по бэклогу
```
1. Открываешь проект «KomekAI» в Cowork (контекст подгружен автоматически)
2. Запускаешь нужный скил: например /marketing:campaign-plan
3. Якорь: «Сделай операционный план Q1 для KomekAI по контексту проекта»
4. Получаешь черновик → правишь итерациями → финализируешь
5. Сохраняешь в Drive или скидываешь себе в репо Marketing-Director
```

### Сценарий Б: текущая задача без скила
```
1. Открываешь проект «KomekAI»
2. Просто пишешь задачу: «Напиши ответ потенциальному клиенту, который
   сомневается, что бот может вести разговор как живой человек»
3. Контекст KomekAI уже у Claude — он сразу применит Caregiver-тон и POD #2
   (казахский голос), не надо напоминать
```

### Сценарий В: ревью готового материала
```
1. Открываешь проект «KomekAI»
2. /marketing:brand-review
3. Вставляешь текст → получаешь чек-лист правок по соответствию
   позиционированию и тону
```

### Сценарий Г: запуск кампании
```
1. /marketing:campaign-plan — план кампании
2. /marketing:content-creation — креативы
3. /marketing:email-sequence — email-аутрич
4. Gmail коннектор → отправка
5. Calendar коннектор → демо-встречи
6. Через 30 дней /marketing:performance-report → отчёт
```

## Когда Cowork лучше Claude Code (и наоборот)

| Cowork лучше | Claude Code лучше |
|---|---|
| Быстрая работа над артефактом без копирования промпта | Работа с файлами / git / репо |
| Использование готовых скилов под типовые задачи | Кастомные итерации, верстка HTML, скрипты |
| Интеграция с Gmail/Calendar/Drive для outbound | Сборка docx через python-docx, работа с большими данными |
| Шаринг с командой через ссылку на проект | Версионирование всего, что делаешь |

**Практика:** используй оба. Cowork — для быстрых артефактов и outbound. Claude Code — для глубокой работы со стратегиями и публикации в репо.
