Прототипы мобильного приложения для сети АЗС **T.S** (Казахстан).

## Где что лежит

| Файл | Что |
|---|---|
| `ТС приложение/TS_app_design_v1..v4.html` | Итеративные версии дизайн-прототипа (v4 — последняя итерация дизайна) |
| `ТС приложение/TS_app_prototype.html` | Финальный кликабельный прототип |
| `ТС приложение/_extracted/tz_*.png` | PNG-фрагменты ТЗ (страница, четверти) — публичные, в репо |
| `ТС приложение/T.S Brandbook.pdf` | **В .gitignore** (`*.pdf`) — оригинальный брендбук, ~32 MB |
| `ТС приложение/_extracted/brandbook/` | **В .gitignore** — 78 MB PNG-извлечений брендбука |
| `ТС приложение/ТЗ для приложения T.S` | **В .gitignore** — PDF ТЗ без расширения |

## Контекст

- Сеть из **16 АЗС** (та же сеть, что и в проекте Receipts Bot — это смежные проекты для одного владельца).
- Дизайн опирается на брендбук T.S и ТЗ заказчика (PDF в локальной папке, в git их нет).
- HTML-прототипы — самостоятельные файлы, можно открывать прямо в браузере.

## Связь с другими проектами

**Receipts Bot** ([project_receipts_bot.md](project_receipts_bot.md)) — тоже про сеть T.S (16 АЗС). Бот собирает чеки от кассиров, приложение — клиентское. Возможна интеграция в будущем (программа лояльности, привязка чеков к клиенту через QR).

## Если работаешь в новой сессии

- Брендбук и ТЗ лежат локально в `ТС приложение/`, в репо их нет — спроси владельца, если нужны исходники.
- При итерациях дизайна сохраняй версионность (`v5`, `v6`), не переписывай предыдущие.
- Прототипы — самодостаточные HTML, без сборки.
